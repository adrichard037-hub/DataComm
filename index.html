Data Communication

A comprehensive overview on course outline

Introduction to Data Communication

Data communication refers to the exchange of data between two or more devices through a transmission medium such as cables or wireless signals.

Components
  • Sender & Receiver: Computers, phones, IoT devices
  • Message: Text, numbers, images, audio, video
  • Medium: Copper cables, fiber optics, air (wireless)
  • Protocol: Defines syntax (format), semantics (meaning), timing (speed/order)
Data Flow Types
  • Simplex: One-way (e.g., keyboard → computer)
  • Half-Duplex: Both directions but not simultaneously (walkie-talkie)
  • Full-Duplex: Simultaneous two-way (phone call)
Performance Metrics
  • Bandwidth: Maximum data capacity (bps)
  • Throughput: Actual data transferred
  • Latency (Delay): Time taken to deliver data
  • Bit Error Rate (BER): Number of corrupted bits

Networking Fundamentals

Networking is the process of connecting two or more computers or devices so they can communicate, share data, and use resources.

Key Features
  • Communication: Devices exchange data (messages, files, videos)
  • Resource Sharing: Share printers, storage, internet
  • Centralized Management: Control systems from one place
  • Scalability: Easily add more devices
Types of Networks
  • LAN: Small area (home, school, office)
  • WAN: Large area (e.g., the internet)
  • MAN: Covers a city
Switching Techniques
  • Circuit Switching: Dedicated path (telephone systems)
  • Packet Switching: Data split into packets (internet)
  • Message Switching: Entire message sent at once (obsolete)
IP Addressing & Ports
  • IPv4: 32-bit (e.g., 192.168.1.1)
  • IPv6: 128-bit (for more devices)
  • HTTP → Port 80 · HTTPS → Port 443 · FTP → Port 21

Network Reference Models

Standard frameworks describing how data moves from one device to another, broken into layers for easier design, understanding, and troubleshooting.

Why They Matter
  • Provide a standard for communication
  • Ensure interoperability between different systems
  • Help troubleshoot network problems
  • Simplify network design and learning
OSI Model (7 Layers)
  • Physical: Bit transmission, cables, voltages
  • Data Link: MAC addressing, error detection (CRC)
  • Network: Logical addressing (IP), routing
  • Transport: Segmentation, flow control, error recovery
  • Session: Establishes, manages, terminates sessions
  • Presentation: Encryption, compression, translation
  • Application: Interfaces (HTTP, FTP, SMTP)
TCP/IP Model (4 Layers)
  • Application: HTTP, DNS, FTP
  • Transport: TCP (reliable), UDP (fast but unreliable)
  • Internet: IP routing
  • Network Access: Physical transmission
Encapsulation
  • Data → Segment → Packet → Frame → Bits

Transmission Media

The physical or wireless path through which data travels from one device to another in a network.

Guided Media
  • Twisted Pair (UTP/STP): Used in LANs & telephone lines. Low cost, easy to install. Limited speed & distance.
  • Coaxial Cable: Central copper core with shielding. Used in cable TV & broadband. Higher bandwidth than twisted pair; bulky & expensive.
  • Fiber Optics: Uses light signals. Single-mode (long-distance) & multi-mode (shorter). Very high speed, immune to EMI; expensive & delicate.
Unguided Media
  • Radio Waves: Omnidirectional. Used in Wi-Fi & mobile networks. Can pass through walls.
  • Microwaves: Require line-of-sight. Used in satellite & long-distance communication. High frequency → higher data capacity.
  • Satellite Signals: Global coverage.

Signals

A signal is a physical representation of data used to transmit information from one device to another through transmission media.

Types of Signals
  • Analog: Continuous waveform (smooth curve). Infinite values within a range. Common in audio & radio.
  • Digital: Discrete values (0 and 1). Represented as square waves. Used in computers & modern communication.
Signal Representation
  • Time Domain: Signal over time
  • Frequency Domain: Signal spectrum
Signal Impairments
  • Noise: Unwanted disturbance causing errors in transmission
  • Attenuation: Loss of signal strength over distance (common in long cables)
  • Distortion: Signal changes shape when different frequencies travel at different speeds
Modulation
  • Analog: AM (Amplitude), FM (Frequency), PM (Phase)
  • Digital: ASK (Amplitude Shift Keying), FSK (Frequency), PSK (Phase)
Data Rate vs Bandwidth
  • Data Rate: Speed of data transmission (bps)
  • Bandwidth: Capacity of the channel — higher bandwidth = more data

Satellite Communication

Transmitting signals over long distances using artificial satellites in space, enabling communication without direct physical connections.

How It Works
  • Signal sent from Earth station to satellite (uplink)
  • Satellite receives & amplifies via a transponder
  • Signal sent back to another Earth station (downlink)
Types of Satellites
  • GEO (36,000 km): Appears stationary. Used for TV broadcasting & weather monitoring.
  • MEO: Between GEO & LEO. Used for GPS navigation.
  • LEO (500–2,000 km): Low latency. Used for modern internet services.
Frequency Bands
  • C-band: Less interference
  • Ku-band: Higher frequency, used in TV
  • Ka-band: High-speed internet
Advantages
  • Covers large geographical areas
  • Useful in remote & rural locations
  • Supports broadcasting & global communication
Disadvantages
  • High cost of launching & maintaining
  • Signal delay (latency), especially GEO
  • Weather interference (rain fade)
Applications
  • TV broadcasting · Internet services · Weather forecasting · GPS & navigation · Military communication

CIA Triad & Cryptography

CIA Triad
  • Confidentiality: Only authorised users can access data. Methods: encryption, passwords & authentication, access control.
  • Integrity: Data is accurate and not altered. Methods: hashing, checksums, digital signatures.
  • Availability: Data & systems accessible when needed. Methods: backups, redundancy systems, disaster recovery plans.
Cryptography

Protecting information by converting it into a secure format so only authorised users can read it.

  • Plaintext: Original readable data
  • Ciphertext: Encrypted (unreadable) data
  • Encryption: Plaintext → Ciphertext
  • Decryption: Ciphertext → Plaintext
Encryption Types
  • Symmetric (AES): One shared key for encryption & decryption. Fast & efficient.
  • Asymmetric (RSA): Public + private key pair.
Other Techniques
  • Hash Functions (SHA-256): One-way, fixed-length output. Used in passwords.
  • Digital Signatures: Verify authenticity and integrity.

Telegraph

A device that transmits coded messages as electrical pulses through wires. Invented by Samuel Morse — the first long-distance electrical communication system.

How It Works
  • Sender presses a telegraph key (switch)
  • Electrical pulses travel through a wire
  • Electromagnet at receiver converts pulses into clicks or marks
  • Message decoded using Morse Code
Morse Code
  • System of dots (·) and dashes (–)
  • A = · –   B = – · · ·   SOS = · · · – – – · · ·
Key Components
  • Telegraph Key: Sends signals
  • Transmission Wire: Carries electrical signals
  • Battery: Provides power
  • Sounder/Receiver: Converts signals into sound
Importance
  • First instant long-distance communication system
  • Replaced slow methods like letters
  • Widely used in military, railways, and business
  • Introduced encoding/decoding, signal transmission & protocols
Limitations
  • Required skilled operators (Morse code)
  • Text-based messages only
  • Limited speed compared to modern systems
